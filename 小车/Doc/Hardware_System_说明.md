## 1. 系统辅助模块 (System)

### 1.1 Delay — 延时函数

| 文件 | `System/Delay.c` / `System/Delay.h` |
|------|--------------------------------------|
| 硬件 | SysTick 系统定时器（芯片内部） |

| 函数 | 说明 |
|------|------|
| `Delay_us(uint32_t xus)` | 微秒级延时，基于 SysTick 实现 |
| `Delay_ms(uint32_t xms)` | 毫秒级延时，调用 `Delay_us(1000)` 循环 |
| `Delay_s(uint32_t xs)` | 秒级延时，调用 `Delay_ms(1000)` 循环 |

**实现原理**：配置 SysTick 的 LOAD 寄存器为 `72 × 微秒数`（72MHz 下每微秒计 72 次），轮询 COUNTFLAG 标志位等待计数完成。此延时为**阻塞式**。

---

### 1.2 TickFlag — 非阻塞延时

| 文件 | `System/TickFlag.c` / `System/TickFlag.h` |
|------|-------------------------------------------|
| 硬件 | TIM1 定时器（芯片内部，1ms 滴答） |

| 函数 | 说明 |
|------|------|
| `Timer_Init(void)` | 初始化 TIM1，周期 1ms，使能更新中断 |
| `TIM1_UP_IRQHandler(void)` | TIM1 中断服务函数，每次溢出 `TickFlag++` |
| `NonBlockingDelay_ms(uint32_t ms)` | 非阻塞毫秒延时，轮询 `TickFlag` 差值 |

**TIM1 配置**：
- `PSC = 71` → 计数频率 = 72MHz / 72 = 1MHz（1μs 计数一次）
- `ARR = 999` → 周期 = 1000 次 = 1ms

## 5. PWM 电机调速模块

| 文件 | `Hardware/PWM.c` / `Hardware/PWM.h` |
|------|--------------------------------------|
| 硬件 | TIM2 定时器通道 3（PA2 引脚输出 PWM） |

| 函数 | 说明 |
|------|------|
| `PWM_Init(void)` | 初始化 TIM2_CH3，PWM 模式 1，频率 20kHz |
| `PWM_SetCompare3(uint16_t Compare)` | 设置 CCR3 值，范围 0~100，控制占空比 |

### PWM 参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 定时器 | TIM2 | 通用定时器 |
| 输出引脚 | PA2 | 复用推挽输出 |
| 预分频 PSC | 35（36-1） | 72MHz / 36 = 2MHz |
| 自动重装 ARR | 99（100-1） | 2MHz / 100 = 20kHz |
| CCR 范围 | 0 ~ 100 | 占空比 = CCR / 100 |
| 输出模式 | PWM1 | 高极性 |

### 接线图

```
STM32F103C8           L298N (ENA)
 ┌─────────┐          ┌──────┐
 │   PA2   ├──────────┤ ENA  │  ← 电机调速 PWM 输入
 └─────────┘          └──────┘
```

> PA2 配置为复用推挽输出 (AF_PP)，输出 PWM 信号至 L298N 的使能引脚 ENA，控制电机转速。

| 文件 | `Hardware/Motor.c` / `Hardware/Motor.h` |
|------|-----------------------------------------|
| 硬件 | L298N 电机驱动板（双路 H 桥） |

| 函数 | 说明 |
|------|------|
| `Motor_Init(void)` | 初始化 PA4~PA7 为推挽输出，调用 `PWM_Init()` |
| `Motor_SetLeftSpeed(int8_t Speed)` | 设置左电机转速和方向，Speed 范围 -100~100 |
| `Motor_SetRightSpeed(int8_t Speed)` | 设置右电机转速和方向，Speed 范围 -100~100 |

### 控制逻辑

| Speed | IN1 (PA4) | IN2 (PA5) | 左电机状态 |
|-------|-----------|-----------|------------|
| > 0   | 1（高） | 0（低） | 正转 |
| = 0   | 1（高） | 1（高） | 刹车 |
| < 0   | 0（低） | 1（高） | 反转 |

| Speed | IN3 (PA6) | IN4 (PA7) | 右电机状态 |
|-------|-----------|-----------|------------|
| > 0   | 1（高） | 0（低） | 正转 |
| = 0   | 1（高） | 1（高） | 刹车 |
| < 0   | 0（低） | 1（高） | 反转 |

> 左右电机共用一个 PWM 信号（PA2 接 L298N ENA），PWM 占空比决定转速。

### 接线图

```
STM32F103C8             L298N 电机驱动板
 ┌──────────┐           ┌──────────────┐
 │    PA2   ├───────────┤ ENA (使能A)  │
 │    PA4   ├───────────┤ IN1          │
 │    PA5   ├───────────┤ IN2          │
 │    PA6   ├───────────┤ IN3          │
 │    PA7   ├───────────┤ IN4          │
 │          │           │              │
 │    VCC   │           │ +12V 电源输入│ ← 外接电池 7~12V
 │    GND   ├───────────┤ GND          │
 └──────────┘           │              │
                        │ OUT1  ├── 左电机+
                        │ OUT2  ├── 左电机-
                        │ OUT3  ├── 右电机+
                        │ OUT4  ├── 右电机-
                        └──────────────┘
```

> L298N 需独立供电（7~12V），STM32 与 L298N 需共地。ENA 接 PWM 信号调速，IN1~IN4 控制方向。

## 9. Ultrasound 超声波测距模块

| 文件 | `Hardware/Ultrasound.c` / `Hardware/Ultrasound.h` |
|------|----------------------------------------------------|
| 硬件 | HC-SR04 超声波测距模块 + TIM4 定时器 |

| 函数 | 说明 |
|------|------|
| `Ultrasound_Init(void)` | 初始化 PB12(Trig) 推挽输出、PB13(Echo) 下拉输入，初始化 TIM4 |
| `Test_Distance(void)` | 触发测距，返回距离（cm），阻塞式 |

### 测距流程

1. PB12 输出 20μs 高电平脉冲（Trig）
2. 等待 PB13（Echo）变为高电平
3. TIM4 开始计数
4. 等待 PB13 变回低电平
5. 读取 TIM4 计数值 → 计算距离

### 距离计算公式

```
距离(cm) = cnt × 0.034 / 2
```

其中 0.034 cm/μs 为声速（340m/s），除以 2 因为声波往返。

### TIM4 配置

| 参数 | 值 | 说明 |
|------|-----|------|
| PSC  | 71（72-1） | 计数频率 1MHz（1μs/次） |
| ARR  | 59999（60000-1） | 最大测量时间 60ms |

### 接线图

```
STM32F103C8            HC-SR04 超声波模块
 ┌─────────┐           ┌──────────┐
 │  PB12   ├───────────┤ Trig     │
 │  PB13   ├───────────┤ Echo     │
 │  VCC    ├───────────┤ VCC (5V) │
 │  GND    ├───────────┤ GND      │
 └─────────┘           └──────────┘
```

> HC-SR04 需 5V 供电，Echo 输出 5V 电平，PB13 配置为下拉输入（IPD），5V 电平对 STM32 的 5V 容忍引脚安全。

## 11. Car 小车运动控制模块

## 12. Tracking 循迹逻辑模块

| 文件 | `System/Tracking.c` / `System/Tracking.h` |
|------|-------------------------------------------|
| 依赖 | Track（传感器）、Car（运动控制） |

| 函数 | 说明 |
|------|------|
| `Tracking(void)` | 双路红外循迹算法，根据左右传感器状态控制小车运动 |

### 循迹逻辑表

| 左传感器 (PB14) | 右传感器 (PB15) | 状态 | 动作 |

## 13. Anticollision 避障逻辑模块

| 文件 | `System/Anticollision.c` / `System/Anticollision.h` |
|------|------------------------------------------------------|
| 依赖 | Ultrasound（测距）、Servo（舵机）、Car（运动控制） |

| 函数 | 说明 |
|------|------|
| `Anticollision(void)` | 超声波避障算法：前方检测障碍 → 左右扫描 → 选择可行方向 |

### 避障流程

```
前进 → 测距(a)
  │
  ├─ a ≥ 15cm → 持续前进
  │
  └─ a < 15cm → 停止 → 舵机转到 0°(右看) → 测距(b)
                   │
                   ├─ b > 15cm → 舵机回中 → 原地右转 → 前进
                   │
                   └─ b ≤ 15cm → 舵机转到 180°(左看) → 测距(c)
                                   │
                                   ├─ c > 15cm → 舵机回中 → 原地左转 → 前进
                                   │
                                   └─ c ≤ 15cm → 舵机回中 → 死循环(无路可走)
```

> 舵机每次测距后延时 1s 等待舵机转动到位。超声波模块安装在舵机上，通过转动舵机实现多方向测距。

## 14. 系统运行流程

```
上电 → 各模块初始化
  │
  ├─ Car_Init()         (电机 + PWM)
  ├─ Servo_Init()       (舵机 PWM)
  ├─ Ultrasound_Init()  (超声波 + TIM4)
  ├─ Infrared_Init()    (循迹传感器)
  ├─ Key_Init()         (按键)
```
