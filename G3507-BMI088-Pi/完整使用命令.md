## 6.1 仅通过串口输出当前姿态角

该模式读取并校准 BMI088，持续解算姿态，不运行 PID，并复用原来的 G3507
10 字节协议帧输出 yaw/pitch 姿态角。

运行：

```bash
cd /home/yinmo/Desktop/BMI088
./build/gimbal_controller --config config/gimbal.conf --attitude-uart
```

启动后的约 2 秒内保持 BMI088 静止。校准完成后 UART 默认以 500 Hz 输出：

```text
5A A5 sequence 02 yaw_L yaw_H pitch_L pitch_H crc_L crc_H
```

完整字段：

```text
第 1–2 字节：5A A5 固定包头
第 3 字节：sequence
第 4 字节：02，姿态模式
第 5–6 字节：yaw × 100，int16 小端
第 7–8 字节：pitch × 100，int16 小端
第 9–10 字节：前 8 字节 CRC16/MODBUS，小端
```

yaw 和 pitch 的单位均为 0.01°。连续 yaw 在发送前折回到 `[-180°, 180°]`，
避免超过 `int16` 范围。该模式不输出 roll。

例如：

```text
yaw = 12.34°  -> 1234  -> D2 04
pitch = -5.67° -> -567 -> C9 FD
```

电脑串口助手设置：

```text
波特率：460800
数据位：8
停止位：1
校验：无
流控：无
显示方式：HEX/十六进制
```

接收端继续使用原 G3507 10 字节二进制协议解析器，通过 `mode=2` 判断当前
第 5–8 字节表示姿态角而不是角速度。串口只输出新鲜、有效的姿态；BMI088
数据超时或姿态无效时不会继续发送旧姿态。

执行 `quit`、按 `Ctrl+C` 或收到 SIGTERM 时，程序会在 mode 2 姿态流结束后
发送 5 个 mode 0 STOP 包，每个包的 yaw/pitch 都为 0。这样即使同一串口实际
连接了电机控制器，也会优先安全停止。纯姿态接收器必须检查 mode，不能把退出
阶段的 STOP 字段当作姿态角。

允许的终端命令：

```text
status
quit
```

以下控制命令在该模式中会被拒绝：

```text
target
run
stop
reset-fault
```

退出：

```text
quit
```

或：

```text
Ctrl+C
```

输出频率继续使用原配置：

```text
uart_frequency_hz = 500
```

`--attitude-uart` 不能与以下选项组合：

```text
--imu-only
--uart-test
--dry-run
--auto-run
--target
```

## 8. 电脑监听发送给 G3507 的原始数据

### 8.1 USB-TTL 监听接线

必须使用支持 3.3 V 电平和 460800 波特率的 USB-TTL。

只连接电脑监听时：

| 树莓派 | USB-TTL |
|---|---|
| Pin 8，GPIO14/TX | RX |
| GND | GND |

USB-TTL 的 TX 不需要连接。树莓派已经单独供电时，不要连接 USB-TTL 的 5 V 或 3.3 V 电源。

同时连接 G3507 和电脑监听时：

```text
树莓派 GPIO14/TX ──┬── G3507 RX
                   └── USB-TTL RX

树莓派 GND ─────────── G3507 GND ─── USB-TTL GND
```

USB-TTL 只作为接收监听器，不要连接它的 TX，避免与其他发送端冲突。

### 8.2 电脑串口助手参数

```text
波特率：460800
数据位：8
停止位：1
校验：无
流控：无
接收显示：HEX/十六进制
```

程序发送的是二进制协议，文本模式会显示乱码。

Windows 可以使用支持 HEX 接收的串口助手、Tera Term、MobaXterm 等工具。先在设备管理器中确认 USB-TTL 对应的 COM 口。

Linux 电脑可以先查找设备：

```bash
ls -l /dev/ttyUSB* /dev/ttyACM*
```

假设设备是 `/dev/ttyUSB0`：

```bash
stty -F /dev/ttyUSB0 460800 cs8 -cstopb -parenb -ixon -ixoff -crtscts raw -echo
xxd -g 1 /dev/ttyUSB0
```

按 `Ctrl+C` 停止抓取。

### 8.3 发送固定测试帧

仅连接电脑、不连接电机控制器时，可以运行：

退出：


## 12. 运行期间交互命令

| 命令 | 作用 |
|---|---|
| `target <yaw> <pitch>` | 设置 yaw/pitch 目标角度，单位为度 |
| `run` | 满足安全条件后进入角度控制 |
| `stop` | 清空控制器状态并发送停止模式 |
| `reset-fault` | 健康帧满足要求后清除锁存故障 |
| `status` | 显示安全状态、目标角度和当前包模式 |
| `quit` 或 `exit` | 安全退出 |

示例：

```text
status
target 15 -3
run
status
stop
quit
```

状态说明：

| 状态 | 含义 |
|---|---|
| `CALIBRATING` | 正在进行 BMI088 静止校准 |
| `STOP_READY` | 传感器正常，但仍保持停止 |
| `RUNNING` | 正在生成有效角速度命令 |
| `DEGRADED` | 姿态或传感器暂时不可信，停止有效输出 |
| `FAULT` | 故障已锁存，需要排查并执行 `reset-fault` |

## 13. G3507 原始协议格式

每帧固定 10 字节，全部多字节字段使用小端序：

| 偏移 | 长度 | 字段 | 说明 |
|---:|---:|---|---|
| 0 | 2 | 帧头 | 固定 `5A A5` |
| 2 | 1 | sequence | 帧序号，自动递增并自然溢出 |
| 3 | 1 | mode | `00`停止、`01`角速度、`02`姿态角、`03`故障 |
| 4 | 2 | yaw | mode 1 为 0.01°/s；mode 2 为 0.01° |
| 6 | 2 | pitch | mode 1 为 0.01°/s；mode 2 为 0.01° |
| 8 | 2 | CRC | CRC-16/MODBUS，小端 |

树莓派代码内部使用中性字段：

```text
GimbalPacketPayload.yaw_value
GimbalPacketPayload.pitch_value
```

按 mode 读取：

```text
getRateControlValues()
getAttitudeValues()
```

MSPM0 端对应使用：

```text
GimbalDecodedPacket.yaw_value
GimbalDecodedPacket.pitch_value
GimbalProtocol_GetRateControl()
GimbalProtocol_GetAttitude()
```

mode 不匹配时 accessor 返回失败，禁止未经 mode 检查直接把字段送进电机速度环。

CRC 参数：

```text
Polynomial = 0xA001
Initial value = 0xFFFF
RefIn = true
RefOut = true
XorOut = 0x0000
计算范围 = 前 8 字节
```

已知完整示例：

```text
5A A5 12 01 E8 09 AE FC 97 73
```

该帧表示：

```text
sequence = 0x12
mode = 1
yaw = 25.36 degree/s
pitch = -8.50 degree/s
CRC = 0x7397
```

默认 `--uart-test` 的前 8 字节类似：

```text
5A A5 XX 01 F4 01 00 00
```

其中：

```text
XX       = 不断变化的帧序号
F4 01    = 小端整数 500，即 5.00 degree/s
00 00    = 0.00 degree/s
最后两字节 CRC 会随序号变化
```

## 15. 常见故障

### 找不到 SPI 设备

```bash
ls -l /dev/spidev*
```

如果不存在，检查 SPI 是否启用，并重启树莓派。

### BMI088 芯片 ID 为 0x00 或 0xFF

检查：

- 3.3 V 供电；
- GND 共地；
- SCLK、MOSI、MISO；
- 加速度计 CS 和陀螺仪 CS；
- 两个 CS 是否接反；
- 模块是否确实使用 BMI088。

### 找不到 `/dev/ttyAMA0`

执行：

```bash
grep -nE 'uart0|uart0-pi5' /boot/firmware/config.txt
ls -l /dev/ttyAMA*
pinctrl get 14 15
```

确认配置后重启：

```bash
sudo reboot
```

### 串口权限不足

```bash
groups
ls -l /dev/ttyAMA0
```

当前用户应属于 `dialout` 组。加入组后需要注销或重启。

### 电脑串口助手显示乱码

确认：

```text
460800 baud
8N1
无流控
HEX 显示
```

原始协议是二进制数据，不应使用文本显示。

### G3507 报 CRC 错误

确认：

- 每帧固定 10 字节；
- 包头在线顺序是 `5A A5`；
- yaw、pitch 和 CRC 都是低字节先发送；
- CRC 初值为 `0xFFFF`；
- CRC 只覆盖前 8 字节；
- 两端均为 460800、8N1、无流控；
- 接收端应通过滑动搜索 `5A A5` 重新同步，不能假设每次中断恰好收到完整一帧。

```text
mspm0_protocol/gimbal_safety_monitor.h
mspm0_protocol/gimbal_safety_monitor.c
```

UART DMA、FIFO 或中断收到任意长度数据时调用：

```c
GimbalStreamParser_Push(&parser, data, length, callback, context);
```

解析器会：

- 滑动搜索 `5A A5`；
- 收集固定 10 字节；
- 验证 mode 和 CRC；
- 错包后只前移一个字节重新同步；
- 处理逐字节、分包、粘包、噪声、缺字节和 CRC 错包；
- 统计输入字节、成功帧、候选错误、CRC 错误、丢弃字节和 sequence 跳变。

安全监控推荐配置：

```c
const GimbalSafetyMonitorConfig monitor_config = {
    .target_timeout_ms = 25U,
    .communication_fault_timeout_ms = 100U
};
```

必须满足：

```text
target_timeout_ms < communication_fault_timeout_ms
```

定期调用：

```c
GimbalSafetyMonitor_Tick(&monitor, monotonic_ms);
```

行为：

- mode 1：更新角速度目标和最后接收时间；
- mode 0：立即归零；
- mode 2：忽略，不进入电机速度闭环；
- mode 3 或未知 mode：锁存故障并归零；
- 25 ms 无新角速度目标：目标归零；
- 100 ms 无合法控制帧：通信故障；
- 恢复后必须收到明确的新 mode 1，不能恢复旧非零目标。

## 16. 推荐实机测试顺序

严格按以下顺序执行：

1. 软件测试：

   ```bash
   ctest --test-dir build --output-on-failure
   ```

2. 只测试 BMI088：

   ```bash
   ./build/gimbal_controller --config config/gimbal.conf --imu-only
   ```

3. 不写串口的完整算法测试：

   ```bash
   ./build/gimbal_controller --config config/gimbal.conf --dry-run
   ```

4. 只连接电脑 USB-TTL，以 HEX 方式检查原 10 字节姿态帧：

   ```bash
   ./build/gimbal_controller --config config/gimbal.conf --attitude-uart
   ```

5. 只连接电脑 USB-TTL，以 HEX 方式监听固定 UART 测试帧：

   ```bash
   ./build/gimbal_controller --config config/gimbal.conf --uart-test
   ```

6. 连接 G3507，但先不接电机或让电机脱离负载，测试停止、运行、断线和 CRC。

7. 最后连接机构，并从很小的目标角度和角速度限制开始测试。

实机测试期间必须具备机械限位、过流/过温保护和可立即操作的急停。MSPM0G3507 端必须独立实现通信超时保护，不能只依赖树莓派程序。
