# 树莓派 5 + BMI088 + MSPM0G3507 云台控制项目 Codex 提示词

你现在运行在树莓派 5 的 Linux 系统上。请在当前目录创建一个完整、可编译、可运行的 C++17 项目，实现 BMI088 姿态解算，并通过 UART 向 TI MSPM0G3507 发送二维云台目标角速度。

当前目录中可能包含以下两个 MSPM0G3507 参考驱动文件：

- `bsp_bmi088.c`
- `bsp_bmi088.h`

请先读取这两个文件，保留其中正确的 BMI088 寄存器地址、初始化顺序、量程配置、原始数据解析、温度换算和单位换算逻辑。

不要直接使用任何 TI MSPM0 DriverLib 函数。以下内容必须改成树莓派 Linux 接口：

- TI SPI 函数改成 Linux `/dev/spidev*` 和 `ioctl`
- TI GPIO 片选函数改成 Linux SPI 硬件片选
- `delay_cycles()` 改成 `std::this_thread::sleep_for()`
- 不使用 Python
- 不使用 wiringPi
- 不依赖 pigpio daemon
- 不使用动态内存完成高频控制循环
- 不在实时循环中频繁输出日志

## 一、硬件连接假设

BMI088 使用 SPI0，共用 SCLK、MOSI、MISO，分别使用两个硬件片选：

- 加速度计：`/dev/spidev0.0`
- 陀螺仪：`/dev/spidev0.1`

默认设置：

- SPI Mode 0
- 8 bit
- 初始速度 1 MHz
- 稳定后允许通过配置文件提高速度
- 加速度计 SPI 读取时必须处理额外的一个 dummy byte
- 陀螺仪读取没有这个额外 dummy byte

UART 使用：

- 默认设备：`/dev/serial0`
- 波特率：460800
- 8N1
- 非阻塞或独立线程发送
- 发送频率：500 Hz
- 目标设备：MSPM0G3507

## 二、BMI088 配置

按照参考代码默认采用：

- 加速度计量程：±6 g
- 陀螺仪量程：±500 °/s

但不要把采样率只限制在原驱动的 100 Hz。请将采样参数集中放在配置文件或常量中，程序初始建议：

- IMU 读取与姿态解算频率：500 Hz
- UART 控制包发送频率：500 Hz

所有寄存器值都要有清楚注释。初始化必须包含：

1. 两个片选释放
2. 加速度计首次读取以切换到 SPI 模式
3. 加速度计软复位
4. 软复位后再次读取以恢复 SPI 模式
5. 陀螺仪软复位
6. 检查加速度计芯片 ID，应为 `0x1E`
7. 检查陀螺仪芯片 ID，应为 `0x0F`
8. 配置加速度计电源
9. 配置加速度计量程和采样参数
10. 配置陀螺仪量程和带宽
11. 对关键寄存器写入后回读校验
12. 初始化失败时输出明确错误原因并安全退出

## 三、树莓派程序需要完成的功能

### 1. BMI088 驱动

实现：

- SPI 设备打开、配置和关闭
- 加速度计寄存器读写
- 陀螺仪寄存器读写
- 芯片 ID 读取
- 原始六轴数据读取
- 温度读取
- 原始值转换为：
  - 加速度：g
  - 角速度：degree/s
  - 温度：摄氏度
- SPI 传输错误检测
- 连续异常数据检测
- RAII 管理文件描述符

驱动接口建议包含：

```cpp
struct ImuSample {
    double timestamp_s;
    float ax;
    float ay;
    float az;
    float gx;
    float gy;
    float gz;
    float temperature_c;
};

class Bmi088 {
public:
    bool initialize();
    bool readSample(ImuSample& sample);
    void close();
};
```

### 2. 零偏校准

程序启动后，云台保持静止，采集大约 2 秒数据，用于计算：

- 陀螺仪 X/Y/Z 零偏
- 加速度计静态均值

校准时：

- 检测设备是否明显运动
- 如果角速度波动过大，提示用户保持静止并重新校准
- 校准结果保存在内存中
- 支持可选地保存到配置文件
- 运行时从测量值中减去陀螺仪零偏

### 3. 坐标轴映射

增加一个独立的坐标变换模块，不要把轴交换和正负号写死在 BMI088 驱动内部。

使用可配置的 3×3 轴映射矩阵，将 BMI088 传感器坐标转换成云台机体坐标。

初始默认使用单位矩阵：

```text
body_x = sensor_x
body_y = sensor_y
body_z = sensor_z
```

在配置文件中允许修改轴顺序和正负方向。

### 4. 数据滤波

实现轻量级低通滤波：

- 加速度计低通
- 陀螺仪低通
- 截止频率可以配置
- 根据真实时间戳计算滤波参数
- 不要假设每次循环的 dt 绝对固定

避免使用过度滤波造成明显控制延迟。

### 5. 姿态解算

实现 Mahony 姿态滤波器：

- 内部使用四元数
- 输入加速度和陀螺仪
- 陀螺仪输入注意从 degree/s 转换为 rad/s
- 使用真实采样时间差 dt
- 对异常 dt 进行限制
- 四元数每次更新后归一化
- 输出 roll、pitch、yaw
- 对外角度单位使用度
- 对 yaw 做连续角度展开，避免从 +180° 突然跳到 -180°

注意：

- BMI088 没有磁力计
- roll 和 pitch 可以由重力方向长期修正
- yaw 只能得到相对角度，会随时间漂移
- 在代码注释和 README 中明确说明 yaw 漂移问题
- 预留以后结合 GM6020 编码器或视觉修正 yaw 的接口

### 6. 云台角度外环

树莓派只运行云台角度外环，不直接计算 GM6020 电流。

输入：

- 当前 yaw 和 pitch
- 目标 yaw 和 pitch

输出：

- yaw 目标角速度
- pitch 目标角速度

分别实现两个 PID 或 PD 控制器，并支持：

- Kp、Ki、Kd 配置
- 输出角速度限制
- 积分限幅
- 抗积分饱和
- 目标角度斜坡
- 角度死区
- yaw 角度跨越处理
- 控制模式切换时清空积分项

树莓派最终只向 MSPM0G3507 发送：

- yaw 目标角速度
- pitch 目标角速度

不要发送电流控制值。

MSPM0G3507 将负责：

- 电机速度闭环
- GM6020 CAN 通信
- 电流指令
- 电机限位
- 过流、过温保护
- 通信失联保护

初始阶段不接入视觉识别，目标角度可以通过：

- 命令行参数
- 配置文件
- 程序运行中的简单终端命令

进行设置。

请预留以后接入 OpenCV 或 YOLO 目标角度的接口，但暂时不要引入 OpenCV 或 YOLO 依赖。

## 四、UART 控制协议

树莓派以 500 Hz 发送固定长度 10 字节二进制包。

包格式严格定义为：

```cpp
#pragma pack(push, 1)
struct GimbalCommandPacket {
    uint16_t header;
    uint8_t sequence;
    uint8_t mode;
    int16_t yaw_rate;
    int16_t pitch_rate;
    uint16_t crc16;
};
#pragma pack(pop)

static_assert(sizeof(GimbalCommandPacket) == 10);
```

字段定义：

```text
偏移 0，2 字节：header，固定为 0xA55A
偏移 2，1 字节：sequence，发送一次加 1，自然溢出
偏移 3，1 字节：mode
偏移 4，2 字节：yaw_rate
偏移 6，2 字节：pitch_rate
偏移 8，2 字节：CRC16
```

模式定义：

```text
0：停止输出
1：角速度控制有效
2：保留
3：故障状态
```

角速度编码：

```text
单位：0.01 degree/s
25.36 degree/s 发送为 2536
-8.50 degree/s 发送为 -850
```

使用 `int16_t`，并在编码前进行范围限制，防止溢出。

协议统一采用小端序。不要依赖 C++ 结构体在不同设备上的直接内存布局，正式发送时请显式逐字节序列化，保证：

```text
header 低字节先发送
所有 int16_t 低字节先发送
CRC 低字节先发送
```

### CRC 要求

实现 CRC16，并在 README 中明确：

- 算法名称
- 多项式
- 初始值
- 是否反转
- 最终异或值
- CRC 覆盖范围

默认使用 CRC-16/MODBUS：

```text
Polynomial：0xA001
Initial value：0xFFFF
RefIn：true
RefOut：true
XorOut：0x0000
```

CRC 覆盖前 8 个字节，不包含最后两个 CRC 字节。

请同时生成 MSPM0G3507 可以直接使用的 C 语言协议头文件，至少包含：

- 包长度
- 包头
- 模式枚举
- CRC16 函数
- 字节解析函数
- int16_t 小端解析方法
- 数据包合法性检查函数

文件名建议：

```text
mspm0_protocol/gimbal_protocol.h
mspm0_protocol/gimbal_protocol.c
```

## 五、串口发送设计

发送周期为 500 Hz，即每 2 ms 生成一次控制包。

要求：

- 不要每次打开和关闭串口
- 程序启动时打开一次
- 配置 termios
- 处理部分写入
- 处理 `EINTR`、`EAGAIN`
- 统计发送失败次数
- 不允许 UART 发送永久阻塞控制循环
- 发送失败时记录计数，但不要在 500 Hz 循环中持续刷屏
- 每秒最多输出一次统计信息

可以使用以下任一设计：

1. 控制线程生成最新指令，UART 线程发送最新指令；
2. 非阻塞 UART 写入加小型单生产者单消费者队列。

优先采用简单、稳定、容易维护的方案。

如果队列堆积，不要发送已经过期的大量旧控制包。应该丢弃旧包，只保留最新控制目标。

## 六、线程和时间调度

建议线程结构：

```text
IMU 线程：
- 500 Hz 读取 BMI088
- 校准
- 坐标转换
- 滤波
- Mahony 姿态更新

控制线程：
- 500 Hz 读取最新姿态
- 运行 yaw 和 pitch 角度外环
- 产生最新目标角速度

UART 线程：
- 500 Hz 发送最新 10 字节控制包

状态线程：
- 1 Hz 输出统计信息
```

也可以在一个高频线程中完成 IMU、姿态和控制，再由独立 UART 线程发送。

要求：

- 使用 `std::chrono::steady_clock`
- 使用绝对时间点调度，避免简单 sleep 造成周期累计漂移
- 统计循环实际频率
- 统计最大周期延迟
- 检测采样超时
- 线程间共享数据必须正确同步
- 尽量避免在高频循环中使用 mutex 长时间阻塞
- 可以使用带序列号的双缓冲或短临界区
- 不做没有必要的复杂无锁框架

程序不需要承诺硬实时。README 中说明普通 Linux 存在调度抖动。

## 七、安全行为

必须实现：

1. 程序刚启动且 BMI088 未初始化成功时，不发送有效运动命令。
2. 校准期间发送停止模式。
3. 姿态数据异常时发送停止模式。
4. SPI 连续读取失败超过阈值时发送停止模式。
5. 控制循环停止更新时，UART 线程不得继续无限发送旧的非零速度。
6. SIGINT 或 SIGTERM 退出时，先连续发送若干个零速度停止包，再关闭串口。
7. yaw 和 pitch 目标角速度都必须有限幅。
8. NaN 或 Inf 绝不能被转换并发送。
9. 所有错误必须返回明确错误信息。
10. 不使用 `sudo` 启动作为程序正常运行的硬性要求；README 说明如何配置 SPI 和串口设备权限。

## 十、构建和运行

使用 CMake，不依赖 ROS。

编译命令应为：

```bash
cmake -S . -B build
cmake --build build -j
```

运行示例：

```bash
./build/gimbal_controller --config config/gimbal.conf
```

支持测试模式：

```bash
./build/gimbal_controller --imu-only
./build/gimbal_controller --uart-test
./build/gimbal_controller --dry-run
```

测试模式说明：

- `--imu-only`：只读取 BMI088 并低频打印姿态，不发送串口。
- `--uart-test`：不依赖 BMI088，发送可配置的固定小角速度，用于示波器或 3507 联调；必须有明确警告和速度限制。
- `--dry-run`：完成全部计算和数据包生成，但不真正写串口。
