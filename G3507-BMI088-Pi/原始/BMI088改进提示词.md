# BMI088 树莓派代码审查与改进提示词

你现在运行在树莓派 5 的 Linux 项目目录中。请先完整阅读当前项目的 `README.md`、`config/gimbal.conf`、`include/`、`src/`、`tests/` 和 `mspm0_protocol/`，然后直接修改、编译并测试。不要只给建议，不要删除已有正确功能，也不要修改 `原始/` 中的参考文件。

## 一、当前审查结论

以下实现已经确认方向正确，修改时应保留：

- BMI088 加速度计和陀螺仪分别使用 `/dev/spidev0.0`、`/dev/spidev0.1`。
- 加速度计读取使用“地址字节 + 额外 dummy byte + 数据”，陀螺仪没有额外 dummy byte。
- 每次寄存器读取通过一次 `SPI_IOC_MESSAGE(1)` 完成，硬件片选不会在地址与数据之间释放。
- ±6 g、±500 degree/s 的原始值换算正确。
- 启动静止校准只把陀螺仪均值作为零偏；代码没有把加速度静态均值从运行数据中减掉，因此没有误删重力。
- 10 字节小端控制包、CRC-16/MODBUS、显式逐字节序列化和 MSPM0 C 解析代码方向正确。
- UART 已使用非阻塞打开，并处理了 `EINTR`、`EAGAIN` 和部分写入。
- 控制命令带更新时间，正常情况下可阻止长期重复发送旧目标。
- 低通滤波按真实 `dt` 计算；循环使用 `steady_clock` 和绝对时间点调度。
- 退出时会尝试发送多个零速度停止包。

不要为了重构而破坏以上行为。

## 二、必须优先修复的问题

### P0-1：校准完成后不得自动启用电机运动

当前 `Application::run()` 在校准成功后执行：

```cpp
control_enabled_.store(!options_.imu_only);
```

这会让普通运行模式在校准结束后立即进入角度外环，并朝默认目标角度运动。尤其 pitch 当前角度不为 0 时，可能突然动作。

请改为：

- 启动、BMI088 初始化、校准完成后仍保持 `STOP`，默认不启用角度外环。
- 只有用户明确输入 `run`，或显式传入一个语义清楚的新参数（例如 `--auto-run`）后才允许 `RATE_CONTROL`。
- 默认配置不得启用 `auto-run`。
- `run` 前检查姿态有效、最近 IMU 数据未超时、UART 已打开（非 `dry-run`）且目标值有限。
- `stop`、故障恢复、模式切换时清空 PID 积分和斜坡状态。
- README 明确写明“程序启动后默认不会驱动电机，需要人工输入 `run`”。
- 添加测试或可测试状态机，验证校准完成不会自动输出非零有效控制包。

### P0-2：修复 UART 部分写入时旧非零帧滞后发送的问题

当前 `uartLoop()` 只有在上一帧完整写完后才检查 `command_timeout_s`。如果一个非零速度帧已经部分写入，而串口持续 `EAGAIN`，该旧帧可能在很久以后才继续发送完成。这样“旧命令超时”不能约束已经进入 `pending` 的帧。

请设计一个有界、可恢复的发送状态机：

- 10 字节帧一旦开始部分写入，不能简单从中间丢弃后直接发送新帧，否则 MSPM0 字节流会错位。
- 记录 pending 帧的创建时间、首次写入时间和模式。
- 使用 `poll(POLLOUT)` 或等价方法，在 UART 线程内用很短的有界等待尽快完成小帧；不得永久阻塞控制线程。
- 为 pending 帧设置严格发送截止时间。超时后进入通信故障状态，清理发送状态，并采用明确的重同步策略；README 说明 MSPM0 接收端必须按双字节包头、固定长度和 CRC 重新同步。
- 串口恢复后首先发送停止帧，不得先补发积压的旧非零控制帧。
- 始终只保留最新尚未开始发送的控制目标，不建立旧命令队列。
- 区分 `EAGAIN`、系统调用错误、帧截止超时和完整帧成功计数。
- sequence 的含义要统一：建议只对“准备发送的新帧”递增，并在 README 说明丢号可能代表本机发送失败或接收丢包；或者改为完整发送后递增，但测试和两端说明必须一致。
- 增加一个不需要真实串口的可注入 writer/fake writer 测试，覆盖：完整写、1+9 部分写、多次 EAGAIN、截止超时、恢复后先发 STOP、无旧非零帧补发。

### P0-3：Mahony 不能在云台运动时无条件信任加速度

当前 `MahonyFilter::update()` 只要加速度模长处于很宽的范围就归一化并全量参与重力校正。云台加速、急停或机体振动时，线加速度会被误当成重力，造成 roll/pitch 瞬时偏转并污染控制目标。

请实现可配置的加速度可信度门控或自适应权重：

- 根据 `|norm(accel)-1g|` 计算可信度。
- 接近 1 g 时正常进行 Mahony 比例/积分校正；偏离阈值时平滑减小权重；严重偏离时只用陀螺仪短时传播姿态。
- 阈值和过渡范围写入配置，例如 `mahony_accel_full_trust_error_g`、`mahony_accel_reject_error_g`。
- 不要在门限处突然开关造成姿态跳变。
- 加速度不可信时冻结或衰减 Mahony 积分项，避免错误积分。
- 给 Mahony 三轴积分状态增加可配置限幅，防止长期 wind-up。
- 对连续加速度拒绝、姿态仅靠陀螺仪传播的持续时间进行统计；超过安全时间可标记姿态降级，但短暂动态加速度不应直接让程序停机。
- 添加 Mahony 单元测试：静止 1 g、短时 1.5 g 线加速度、零加速度、NaN、异常 `dt`、积分限幅、四元数持续归一化。

### P0-4：修正信号处理方式

当前 POSIX 信号处理函数通过全局 `Application*` 调用 C++ 对象方法和 `std::atomic<bool>::store()`。这不应被假定为异步信号安全。

请改为安全方案：

- 信号处理函数只设置 `volatile std::sig_atomic_t` 标志，或使用 Linux `signalfd`/self-pipe 方案。
- 正常线程观察到该标志后再调用 `requestStop()`。
- 信号处理函数中不得使用 iostream、分配内存、锁 mutex 或调用复杂对象方法。
- 保留 SIGINT/SIGTERM 下发送停止包并关闭设备的正常退出流程。

## 三、树莓派 5 UART 配置必须纠正

当前 README 和默认配置把 GPIO14/GPIO15 与 `/dev/serial0` 直接视为同一个 UART，这在树莓派 5 上不能默认成立。Pi 5 的 `/dev/serial0` 常指向独立调试 UART，而不一定是 40 针排针的物理引脚 8/10。

请修改 README 和配置示例：

- 对树莓派 5 的 GPIO14/GPIO15 方案，说明需要在 `/boot/firmware/config.txt` 启用适用 overlay，例如 `dtoverlay=uart0-pi5`，然后重启。
- 删除串口登录控制台占用，但保持 `/boot/firmware/cmdline.txt` 单行格式。
- 要求用户在实机执行：

```bash
readlink -f /dev/serial0
ls -l /dev/ttyAMA*
pinctrl get 14
pinctrl get 15
```

- 程序配置必须使用实际映射到 GPIO14/GPIO15 的节点；示例可写 `/dev/ttyAMA0`，但必须注明以实机查询结果为准。
- 不要声称 `/dev/serial0` 在所有 Pi 5 上都对应 GPIO14/GPIO15。
- 建议将 `config/gimbal.conf` 的 Pi 5 40 针示例改成 `/dev/ttyAMA0`，同时在 README 给出如何确认和修改。
- 程序启动时打印“打开的串口路径”；若路径是 `/dev/serial0`，尽可能解析并打印其真实目标，帮助排错。

## 四、建议同步完成的控制改进

### P1-1：PID 的 D 项优先使用测量角速度

当前 D 项使用相邻角度误差差分。它容易放大姿态噪声，并在目标变化时产生 derivative kick。项目已经有滤波后的 BMI088 角速度，适合二维云台外环使用。

请将控制器扩展为可选择“对测量求导”或直接使用对应机体轴角速度作为阻尼反馈：

```text
rate_cmd = Kp * angle_error + Ki * integral - Kd * measured_rate
```

要求：

- yaw/pitch 对应哪个机体角速度轴必须可配置，不能假定传感器原始 X/Y/Z。
- 轴映射后再选控制轴和符号。
- 保留 PID 输出限幅、积分限幅、抗饱和、死区和目标斜坡。
- 默认参数保持保守，并在 README 说明调参顺序：先 Kp，再 Kd，最后必要时才加 Ki。
- 添加目标阶跃无 D 项冲击、测量角速度阻尼方向正确的测试。

### P1-2：故障状态应有锁存和明确恢复条件

当前单次 SPI 失败就发布 `FAULT`，下一次成功采样又可能立即恢复有效控制，模式可能快速抖动。

请实现简单状态机：

```text
BOOT/CALIBRATING -> STOP_READY -> RUNNING -> DEGRADED/FAULT
```

- 单次偶发 SPI 失败：本周期输出 STOP 或保持安全策略，但统计错误；不要无条件自动恢复成 RUNNING 而没有连续成功样本确认。
- 达到连续失败阈值：锁存 FAULT，必须连续成功若干帧并由用户重新 `run`，或明确 `reset-fault` 后恢复。
- 校准失败、姿态无效、UART 发送截止超时分别记录故障原因。
- 状态输出显示当前状态、最后故障和恢复条件。
- 3507 仍必须独立执行 20~30 ms 目标降速、约 100 ms 通信故障和最终电机保护；树莓派不能替代 MCU 安全保护。

### P1-3：配置验证补强

请增加以下验证：

- 低通截止频率必须小于对应循环 Nyquist 频率，并给出合理余量。
- `command_timeout_s` 必须大于至少若干个 UART 周期，避免正常抖动误判，同时不能无限大。
- PID 的 Kp/Ki/Kd 必须为有限非负值；所有 float/double 配置必须拒绝 NaN/Inf。
- Mahony 阈值、积分限幅、故障恢复连续成功帧数必须合法。
- UART 频率、10 字节帧、8N1 与波特率之间做带宽检查，并保留安全余量。
- SPI 配置读取回来的实际速度如果与请求值不同，应记录实际值，而不是只检查 mode 和 bits。
