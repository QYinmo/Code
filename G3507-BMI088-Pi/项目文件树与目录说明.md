## 根目录

- `CMakeLists.txt`：构建主程序、可移植 MSPM0 C 库和全部单元测试；支持
  `GIMBAL_WARNINGS_AS_ERRORS=ON` 严格告警模式。
- `README.md`：设计、接线、协议、安全边界和验证说明。
- `完整使用命令.md`：面向实际操作的完整命令手册。
- `项目文件树与目录说明.md`：本文件。

`build/` 和 `build-strict/` 是 CMake 生成目录，不属于源代码。

## config

- `gimbal.conf`：SPI/UART 设备、频率、滤波器、Mahony、PID、校准和安全超时配置。

## include 与 src

- `application.*`：硬件生命周期、校准、控制循环、UART/状态/输入线程和安全退出。
- `bmi088.*`：Linux spidev BMI088 驱动，分别处理加速度 dummy byte 和陀螺仪读取。
- `config.*`：配置加载和边界校验。
- `gimbal_protocol.*`：树莓派端 10 字节协议序列化、CRC、mode 语义和安全 accessor。
- `run_mode_policy.*`：普通模式/姿态模式的纯决策逻辑，包括姿态发布、超时抑制、
  命令权限、UART 故障动作和退出 STOP payload。
- `mahony_filter.*`：姿态解算、加速度可信度门控、积分衰减和 yaw 连续展开。
- `pid_controller.*`：yaw/pitch 角度外环。
- `safety_state.*`：BOOT、CALIBRATING、STOP_READY、RUNNING、DEGRADED、FAULT。
- `uart_port.*`：Linux 非阻塞 UART。
- `uart_tx_state_machine.*`：部分写、EAGAIN、帧截止时间和恢复 STOP。
- `signal_stop.*`：异步信号安全的 SIGINT/SIGTERM 停止标志。
- `exit_status.hpp`：进程退出原因和退出码优先级。

## mspm0_protocol

该目录可以复制到 MSPM0G3507 工程：

- `gimbal_protocol.*`：整包校验、解析和 mode 安全 accessor；
- `gimbal_stream_parser.*`：任意分块连续字节流解析、噪声跳过和滑动重同步；
- `gimbal_safety_monitor.*`：STOP/FAULT/mode 分发、25 ms 目标超时和 100 ms 通信故障；
- `README.md`：接入示例和必须由真实电机工程实现的保护。

这些文件只依赖标准 C 头文件，不依赖 Linux 或 MSPM0 SDK。

## tests

- `test_protocol`：树莓派协议、CRC、字段编码和 mode 安全 accessor。
- `test_attitude_uart`：姿态模式发布条件、超时、命令隔离、UART 故障策略和退出 STOP。
- `test_mspm0_stream_parser`：任意分块、噪声、粘包、错位、CRC 恢复、包头和 sequence。
- `test_mspm0_safety`：MSPM0 mode 分发、目标超时、通信故障、恢复和故障锁存。
- 其他测试覆盖 BMI088 换算、Mahony、PID、UART 状态机、安全状态、配置、信号和退出码。

所有测试均不访问真实 BMI088、UART、电机或真实时间睡眠。
