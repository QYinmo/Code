## 二、系统全局数据结构 (`SystemData_t`)

|----------|----------|------|
| 状态机 | `state` | `SystemState_t` 枚举 |
| 室内传感器 | `ambient_temp/humi`, `indoor_abs_humi`, `indoor_aht20_ok` | AHT20 #1 via I2C1 |
| 室外传感器 | `outdoor_temp/humi`, `outdoor_abs_humi`, `outdoor_sensor_ok` | AHT20 #2 via I2C2 |
| 红外传感器 | `ir_ambient_temp`, `ir_object_temp`, `mlx90614_ok/obj_ok` | MLX90614 via I2C1 |

| PWM | `pwm_duty` | 0 ~ 2879 (ARR) |

| 按键 | `key_event` | SysTick 中断设置, 主循环消费 |

## 三、核心状态机逻辑

状态机在 `App_UpdateStateMachine()` 中实现，按优先级从高到低：

```
SYS_STATE_PREHEAT       (上电 → MQ-2 预热 20s)
       ↓ (预热完成)
SYS_STATE_GAS_ALARM     ← 优先级 1: 气体报警, 全速排风
       ↓ (解除)
SYS_STATE_STALL_FAULT   ← 优先级 2: 堵转故障, 停机保护
       ↓ (KEY1 复位)
SYS_STATE_HOT_WATER     ← 优先级 3: 热水运行, PID 闭环调速
       ↓ (热水停止)
SYS_STATE_DELAY_STOP    ← 优先级 4: 延时除湿 (PID + 绝对湿度收敛)
       ↓ (收敛/超时)
SYS_STATE_IDLE          ← 空闲, 风扇停止
```

### 1. 预热阶段 (`PREHEAT`)
- 系统上电进入, MQ-2 预热 20 秒
- 黄灯慢闪, 其余功能正常响应
- 预热结束后自动进入 `IDLE`

### 2. 气体报警 (`GAS_ALARM`) — 最高优先级
- 条件: MQ-2 ADC >= 2200 连续 3 次 (迟滞释放阈值 1800, 需连续 5 次低于才解除)
- 动作: 风扇 100% 全速, 红灯闪烁, 蜂鸣器 200ms 间隔急促鸣叫
- 无视热水状态和延时状态

### 3. 堵转故障 (`STALL_FAULT`) — 第二优先级
- **双判据联合检测** (任一满足即触发):
  - 判据 1: ACS712 检测电流超过自适应阈值, 持续 > 500ms
  - 判据 2: PWM > 30% 但 RPM < 100, 持续 > 3000ms
- 动作: 立即停机, 红灯 + 蜂鸣器 500ms 间隔慢报警
- 需按 KEY1 手动复位

### 4. 热水运行 (`HOT_WATER`) — 双通道 max + 斜率限制
- 触发条件: MLX90614 水温 - AHT20 室温 >= 5℃ (迟滞关闭阈值 3℃)
- **调速算法: 温度/湿度双通道 + max + 斜率限制**
  - 温度通道: 水温-室温温差 线性映射 (ΔT 5℃→30%, 25℃→90%) + 室温修正
  - 湿度通道: PID 闭环 (PV=室内绝对湿度, SP=室外绝对湿度)
  - 目标占空比 = max(温度需求, 湿度需求), 谁更急谁主导
  - 斜率限制: 升速快 (≈2.8%/cycle), 降速慢 (≈1.0%/cycle)
- 最低占空比保障: 30% (防止蒸汽未达传感器)
- 低温保护: 室温 < 18℃ 时限制最大占空比 50%
- 室外传感器离线时湿度通道降级为固定 40% 保守模式

### 5. 延时除湿 (`DELAY_STOP`) — 湿度主导 + 温度衰减
- 热水停止后进入, 分两段:
  - **基础延时** (默认 30s, 可按键调节 10~120s): 无条件运行
  - **收敛延时** (基础延时后): 监测室内外绝对湿度差
    - 差值 < 2 g/m³ → 收敛完成, 停机
    - 超过强制超时 (延时 × 3) → 强制停机
- **调速策略**: 湿度通道主导, 温度通道随时间衰减
  - 温度衰减系数 = 1 - elapsed/DELAY_STOP_DECAY_MS (从 1.0 → 0.0)
  - target = max(湿度需求, 温度需求 × 衰减)
  - 同样经斜率限制后输出
- 最低占空比降为基础值的一半

## 四、重点算法 1: 绝对湿度 PID 闭环

### 绝对湿度计算 (Magnus-Tetens 公式)

```
饱和蒸气压: Es = 6.112 × exp(17.67 × T / (T + 243.5))  [hPa]
实际蒸气压: Ea = Es × RH / 100
绝对湿度:   AH = 216.7 × Ea / (T + 273.15)              [g/m³]
```

实现在 `bsp_humidity.c` 的 `Humidity_CalcAbsolute()` 函数中。

### PID 控制器

实现在 `bsp_pid.c`，核心特性:
- **反向作用**: `error = measured - setpoint` (室内湿度越高于室外, 输出越大)
- **真实 dt**: 使用 `HAL_GetTick()` 计算实际时间间隔, 不受主循环速度波动影响
- **Anti-Windup**: 积分限幅 = (output_max - output_min) × 0.6
- **dt 钳位**: 最小 10ms (防除零), 最大 2s (防长时间挂起后积分暴增)

默认参数: Kp=200, Ki=5, Kd=50, 可通过 Flash 持久化存储。

## 五、重点算法 2: 热水判定 (迟滞比较)

```c
delta = T_water(MLX90614 Tobj) - T_ambient(AHT20 室温)

开启阈值: delta >= 5.0℃ → hot_water = true
关闭阈值: delta <  3.0℃ → hot_water = false
```

- MLX90614 BAA 90° 宽视场角: 花洒移动时仍能检测
- 滑动平均滤波 (窗口=5): 消除快速移动抖动
- 需要 MLX90614 Tobj 和室内 AHT20 同时在线才判定

## 六、重点算法 3: 堵转检测 (电流 + RPM 联合)

### ACS712 自适应校准
1. **上电零点校准**: 风扇静止时采样 50 次取平均 → `acs712_zero_adc`
2. **启动峰值自适应**: 风扇首次启动后 2s 内记录峰值电流偏差 → 堵转阈值 = 峰值 × 5/2

### 堵转检测 (电流 + RPM 联合, 4线风扇调整)
- PA7 输入捕获, 下降沿触发
- RPM = 60 × 1000000 / (capture_diff × 2)
- 超时 2s 无脉冲 → RPM = 0

### 判据逻辑 (PWM > 30% 时启用)
```
判据1: |current_adc - zero_adc| > stall_threshold 持续 > 800ms  (4线风扇调整)
判据2: fan_rpm < 100 持续 > 2000ms                            (4线风扇调整)

任一满足 → 堵转故障
```

## 七、重点算法 4: Flash 参数持久化

利用 STM32F103C8T6 内部 Flash 最后一页 (Page63, 0x0800FC00) 存储:
- PID 参数 (Kp, Ki, Kd)
- 延时时间 (fan_delay_sec)
- ACS712 零点校准值

特性:
- 魔数校验 (0xA5C3E1D7): 首次使用/数据损坏时自动使用默认值
- 延迟写入: 按键修改参数后等待 3s 无新操作才写入 (减少 Flash 磨损)
- 上电加载时做范围校验, 防止异常数据

## 八、传感器降级策略

系统对每个传感器独立管理在线状态, 实现优雅降级:

| 传感器 | 降级条件 | 降级策略 |
|--------|----------|----------|
| 室内 AHT20 | 连续失败 10 次 或 数据超时 2s | 禁止热水联动判定 |
| 室外 AHT20 | 连续失败 10 次 | 使用经验默认值 (20℃, 50%RH); 触发 I2C2 总线恢复 |
| MLX90614 Ta | 连续失败 10 次 或 数据超时 2s | 标记离线 |
| MLX90614 Tobj | 连续失败 10 次 或 数据超时 2s | 禁止热水联动判定 |
| 传感器降级时 | — | PID 关闭, 风扇固定 40% 占空比保守运行 |

### I2C2 总线死锁恢复
室外 AHT20 连续失败达阈值时, 自动执行:
1. 关闭 I2C2 外设
2. SCL/SDA 切为开漏 GPIO
3. 发送 9 个时钟脉冲 + STOP 条件
4. 重新初始化 I2C2 + AHT20

## 九、按键处理机制

采用 **SysTick 中断扫描** (非 EXTI), 在 `stm32f1xx_it.c` 中实现:
- 扫描周期: 10ms (SysTick 每 1ms 中断, 每 10 次执行一次扫描)
- 消抖: 连续 3 次 (30ms) 确认
- 长按连发: 500ms 后开始, 200ms 间隔
- 主循环通过关中断原子读取 `key_event` 标志

| 按键 | 引脚 | 功能 | 特性 |
|------|------|------|------|
| KEY1 | PA4 | 故障复位 | 仅单击, 不连发 |
| KEY2 | PA5 | 延时 +10s | 支持长按连发 |
| KEY3 | PA6 | 延时 -10s | 支持长按连发 |

## 十、OLED 显示内容 (7 行, 200ms 刷新)

```
第0行: MODE:HOT WATER    或 MODE:IDLE / PREHEAT / GAS ALARM ...
第1行: In:25.3C 58%      室内温湿度 + 传感器状态(! = 离线)
第2行: Out:22.1C 45%     室外温湿度 (离线时显示 Out:OFFLINE)
第3行: dAH: 3.2 Tw:45.1  绝对湿度差 + 水温 + MLX状态
第4行: P:52% R:1200 dT:20 占空比 + 转速 + 温度通道ΔT
第5行: Gas:1234 I:0.35A  MQ-2 ADC + 风扇电流
第6行: B:25s F:65s H:HI  基础/强制倒计时 + 湿度收敛状态
```

## 十一、主循环结构

```c
while (1) {
    App_HandleKeys();          // 消费按键事件
    App_ReadSensors();         // 非阻塞传感器采样
    App_UpdateStateMachine();  // 状态机更新
    App_ExecuteOutput();       // PWM/LED/蜂鸣器 + 喂狗
    App_UpdateDisplay();       // OLED 刷新 (200ms 限速)
    App_FlashDeferredSave();   // Flash 延迟保存
    App_ACS712_PeakTrack();    // ACS712 启动峰值学习
}
```

无 `HAL_Delay()`, 全部非阻塞设计, 主循环自由运行。IWDG 看门狗 4 秒超时, 在 `App_ExecuteOutput()` 中喂狗。
