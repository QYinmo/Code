## 二、方式 A：基于 STM32CubeMX 新建工程（推荐）

### 步骤 1：CubeMX 配置

1. 打开 STM32CubeMX，新建 STM32F103C8Tx 工程
2. 按以下参数配置外设：

| 外设 | 参数 | 引脚 |
|------|------|------|
| RCC | HSE = Crystal | PD0/PD1 |
| SYS | Debug = Serial Wire | PA13/PA14 |
| TIM1 CH1 | PWM Generation, ARR=2879, PSC=0 | PA8 |
| TIM3 CH2 | Input Capture, 下降沿, PSC=71 | PA7 |
| ADC1 IN0 | Single Conversion | PA0 |
| ADC1 IN1 | Single Conversion | PA1 |
| I2C1 | Standard 100kHz | PB6(SCL)/PB7(SDA) |
| I2C2 | Standard 100kHz | PB10(SCL)/PB11(SDA) |
| USART1 | 115200 8N1 | PA9(TX)/PA10(RX) |

3. GPIO 手动配置：
   - PB0/PB1/PB2/PB5 → Output Push-Pull, 默认高电平 (LED/蜂鸣器低电平触发)
   - PA4/PA5/PA6 → Input Pull-up
   - 为引脚设置 User Label: KEY1, KEY2, KEY3, LED_GREEN, LED_YELLOW, LED_RED, BUZZER

4. NVIC 配置：
   - **TIM3 global interrupt → Enable** (风���测速输入捕获必需)

5. Clock Configuration：
   - HSE 8MHz → PLL ×9 → SYSCLK 72MHz
   - APB1 = 36MHz, APB2 = 72MHz
   - ADC Prescaler = /6 → 12MHz

6. Project Manager：
   - Toolchain: MDK-ARM V5 (或 STM32CubeIDE)
   - 勾选 "Generate peripheral init as pair of .c/.h"

7. 点 **GENERATE CODE**

### 步骤 2：复制用户代码文件

CubeMX 生成代码后，将本项目的 BSP 文件复制到工程中：

```bash
# 复制头文件到 Core/Inc/
bsp_config.h
bsp_aht20.h
bsp_mlx90614.h
bsp_oled.h
bsp_pid.h
bsp_humidity.h
bsp_flash_storage.h
bsp_fan_tacho.h

# 复制源文件到 Core/Src/
bsp_aht20.c
bsp_mlx90614.c
bsp_oled.c
bsp_pid.c
bsp_humidity.c
bsp_flash_storage.c
bsp_fan_tacho.c
```

### 步骤 3：编辑 CubeMX 生成的 main.c

打开 CubeMX 生成的 `main.c`，在 `/* USER CODE BEGIN */` 注释区域内，
把本项目 `main.c` 中的以下内容**复制粘贴**进去：

- **头文件区**（`USER CODE BEGIN Includes`）：
  ```c
  #include "bsp_config.h"
  #include "bsp_aht20.h"
  #include "bsp_mlx90614.h"
  #include "bsp_oled.h"
  #include "bsp_pid.h"
  #include "bsp_humidity.h"
  #include "bsp_flash_storage.h"
  #include "bsp_fan_tacho.h"
  #include <stdio.h>
  #include <string.h>
  #include <math.h>
  #include <stdarg.h>
  ```

- **全局变量区**（`USER CODE BEGIN PV`）：
  ```c
  SystemData_t g_sys = {
      .state = SYS_STATE_PREHEAT,
      .fan_delay_sec = FAN_DELAY_DEFAULT,
      .acs712_zero_adc = ACS712_ZERO_CURRENT_ADC_DEFAULT,
      .acs712_stall_threshold = ACS712_STALL_THRESHOLD_DEFAULT,
  };

  static PID_Controller_t g_pid;
  static uint8_t outdoor_fail_count = 0;
  static AHT20_Handle_t h_aht20_indoor;
  static AHT20_Handle_t h_aht20_outdoor;
  static uint8_t indoor_aht20_fail_cnt = 0;
  ```

- **函数声明区**（`USER CODE BEGIN PFP`）：
  复制所有 `static void App_XXX()` / `static xxx Detect_XXX()` 等函数声明

- **while(1) 之前**（`USER CODE BEGIN 2`）：
  ```c
  App_Init();
  ```

- **while(1) 内部**（`USER CODE BEGIN 3`）：
  ```c
  App_HandleKeys();
  App_ReadSensors();
  App_UpdateStateMachine();
  App_ExecuteOutput();
  App_UpdateDisplay();
  App_FlashDeferredSave();
  App_ACS712_PeakTrack();
  ```

- **文件末尾**（`USER CODE BEGIN 4`）：
  粘贴所有 `App_XXX` / `Detect_XXX` / `Fan_XXX` / `LED_Set` 等函数实现

### 步骤 4：编辑 stm32f1xx_it.c

在 CubeMX 生成的 `stm32f1xx_it.c` 中添加：

- **头文件区**（`USER CODE BEGIN Includes`）：
  ```c
  #include "tim.h"
  #include "bsp_fan_tacho.h"
  #include "bsp_config.h"
  ```

- **SysTick_Handler 内**（`USER CODE BEGIN SysTick_IRQn 1`）：
  复制本项目的按键扫描代码块

- **文件末尾**（`USER CODE BEGIN 1`）：
  ```c
  void TIM3_IRQHandler(void)
  {
    HAL_TIM_IRQHandler(&htim3);
  }

  void HAL_TIM_IC_CaptureCallback(TIM_HandleTypeDef *htim)
  {
    FanTacho_CaptureCallback(htim);
  }
  ```

### 步骤 5：在 Keil 中添加源文件

CubeMX 只会将它自己生成的 .c 文件注册到 Keil 工程中，用户自定义的 bsp_xxx.c 不会自动加入：

1. Keil → **Project → Manage Project Items** (或右键 Source Group)
2. 在 `Core/Src` 分组里点 **Add Files**
3. 添加以下 7 个文件：
   - `bsp_aht20.c`
   - `bsp_mlx90614.c`
   - `bsp_oled.c`
   - `bsp_pid.c`
   - `bsp_humidity.c`
   - `bsp_flash_storage.c`
   - `bsp_fan_tacho.c`

4. 确认 Include Paths 包含 `Core/Inc` (CubeMX 通常已自动配置)

### 步骤 6：编译设置

1. **Options → Target → Use MicroLIB** → 勾选 (支持 snprintf 浮点格式化)
2. **Options → C/C++ → Define**: 确认包含 `USE_HAL_DRIVER,STM32F103xB`
3. **Options → C/C++ → Include Paths**: 确认包含：
   ```
   Core/Inc
   Drivers/STM32F1xx_HAL_Driver/Inc
   Drivers/CMSIS/Device/ST/STM32F1xx/Include
   Drivers/CMSIS/Include
   ```

### 步骤 7：编译下载

1. Build (F7) → 应无错误
2. Download (F8) → 通过 ST-Link

## 三、方式 B：不用 CubeMX，纯 Keil 手动建工程

### 步骤 1：获取 HAL 库

从 ST 官网下载 STM32CubeF1 固件包（或从 CubeMX 安装目录复制）：

```
STM32Cube_FW_F1_Vx.x.x/
├── Drivers/
│   ├── STM32F1xx_HAL_Driver/    ← HAL 库源码
│   │   ├── Inc/
│   │   └── Src/
│   └── CMSIS/
│       ├── Device/ST/STM32F1xx/ ← 设备头文件 + startup
│       └── Include/             ← CMSIS Core
```

### 步骤 2：Keil 新建工程

1. Project → New µVision Project → 选择 STM32F103C8
2. 不要使用 Manage Run-Time Environment（取消勾选）

### 步骤 3：添加源文件分组

| 分组名 | 文件 |
|--------|------|
| Application | `main.c`, `stm32f1xx_hal_msp.c`, `stm32f1xx_it.c`, `system_stm32f1xx.c` |
| CubeMX_Periph | `adc.c`, `gpio.c`, `i2c.c`, `tim.c`, `usart.c` |
| BSP_Drivers | `bsp_aht20.c`, `bsp_mlx90614.c`, `bsp_oled.c`, `bsp_pid.c`, `bsp_humidity.c`, `bsp_flash_storage.c`, `bsp_fan_tacho.c` |
| HAL_Driver | 从 HAL 库 Src/ 中添加以下文件: |
| | `stm32f1xx_hal.c` |
| | `stm32f1xx_hal_cortex.c` |
| | `stm32f1xx_hal_rcc.c`, `stm32f1xx_hal_rcc_ex.c` |
| | `stm32f1xx_hal_gpio.c`, `stm32f1xx_hal_gpio_ex.c` |
| | `stm32f1xx_hal_adc.c`, `stm32f1xx_hal_adc_ex.c` |
| | `stm32f1xx_hal_i2c.c` |
| | `stm32f1xx_hal_tim.c`, `stm32f1xx_hal_tim_ex.c` |
| | `stm32f1xx_hal_uart.c` |
| | `stm32f1xx_hal_dma.c` |
| | `stm32f1xx_hal_flash.c`, `stm32f1xx_hal_flash_ex.c` |
| | `stm32f1xx_hal_pwr.c` |
| | `stm32f1xx_hal_exti.c` |
| Startup | `startup_stm32f103xb.s`（从 CMSIS 复制） |

### 步骤 4：配置 Include 路径

Options for Target → C/C++ → Include Paths：

```
Core/Inc
Drivers/STM32F1xx_HAL_Driver/Inc
Drivers/CMSIS/Device/ST/STM32F1xx/Include
Drivers/CMSIS/Include
```

### 步骤 5：配置预处理宏

Options for Target → C/C++ → Define：

```
USE_HAL_DRIVER,STM32F103xB
```

### 步骤 6：配置目标芯片

Options for Target → Target：
- IROM1: 0x08000000, Size 0x10000 (64KB Flash)
- IRAM1: 0x20000000, Size 0x5000  (20KB RAM)
- 勾选 **Use MicroLIB**

Options for Target → Output：
- 勾选 Create HEX File

### 步骤 7：编译并下载

1. F7 编译 → 无错误
2. F8 下载 → 选择 ST-Link

## 四、常见编译错误排查

| 错误信息 | 原因 | 解决 |
|----------|------|------|
| `stm32f1xx_hal_conf.h not found` | Include 路径未添加 Core/Inc | 添加到 Include Paths |
| `undefined symbol SystemInit` | 未添加 system_stm32f1xx.c | 加入工程编译 |
| `undefined symbol HAL_IncTick` | 未添加 stm32f1xx_hal.c | 添加 HAL 基础库文件 |
| `undefined symbol PID_Init` | 未添加 bsp_pid.c 到工程 | Manage Project Items → 添加 |
| `undefined symbol Humidity_CalcAbsolute` | 未添加 bsp_humidity.c 到工程 | Manage Project Items → 添加 |
| `undefined symbol Flash_Load` | 未添加 bsp_flash_storage.c | Manage Project Items → 添加 |
| `undefined symbol FanTacho_Init` | 未添加 bsp_fan_tacho.c | Manage Project Items → 添加 |
| `Error_Handler undefined` | main.h/main.c 中未定义 | 确认 main.c 末尾有 Error_Handler() |
| `HAL_TIM_MspPostInit undefined` | 未添加 stm32f1xx_hal_msp.c | 加入工程编译 |
| `SysTick_Handler undefined` | 未添加 stm32f1xx_it.c | 加入工程编译 |
| `HAL_Delay 死锁不返回` | SysTick_Handler 未正确实现 | 确保 stm32f1xx_it.c 中调用了 HAL_IncTick() |
| `L6218E: __use_no_semihosting` | 使用了 printf 但未配置 MicroLIB | 勾选 Options→Target→Use MicroLIB |
| `浮点 snprintf 无输出` | MicroLIB 不支持 %f | 勾选 Use MicroLIB |
| `expf undefined` | 未链接数学库 | MicroLIB 已包含; 或在 Linker 加 `-lm` |

## 五、CubeMX 重新生成代码后的注意事项

CubeMX 重新生成代码时会覆盖它管理的文件，但会保留 `USER CODE BEGIN/END` 区域：

**会被覆盖的文件** (但 USER CODE 区域保留):
- `main.c`, `main.h`
- `adc.c/h`, `gpio.c/h`, `i2c.c/h`, `tim.c/h`, `usart.c/h`
- `stm32f1xx_it.c/h`, `stm32f1xx_hal_msp.c`

**不会被覆盖的文件** (CubeMX 不管理):
- 所有 `bsp_*.c/h` 文件
- `bsp_config.h`

**重要**: 重新生成后:
1. 检查 `.mxproject` 文件, 确认 CubeMX 没有移除用户文件
2. 在 Keil 中确认 `bsp_*.c` 文件仍在工程项目组中
3. 如果 CubeMX 升级了 HAL 库版本, 注意 API 兼容性

## 六、STM32CubeIDE 用户

如果使用 STM32CubeIDE 而非 Keil：

1. CubeMX 中选择 Toolchain = STM32CubeIDE
2. 生成代码后按方式 A 的步骤 2/3/4 操作
3. 右键工程 → Properties → C/C++ Build → Settings
4. 确认 Include Paths 包含 Core/Inc
5. 确认所有 `bsp_*.c` 文件在 Source 列表中 (CubeIDE 会自动发现 Core/Src 下的 .c)
6. Build Project (Ctrl+B)
7. Run → Debug As → STM32 Cortex-M C/C++ Application

## 七、CubeMX .ioc 当前状态与代码的差异

**当前 SmartFan.ioc 缺少以下配置** (代码中已使用但 .ioc 中未配):

| 缺少的配置 | 代码中使用位置 | 影响 |
|------------|---------------|------|
| I2C2 (PB10/PB11) | `i2c.c`: MX_I2C2_Init(), 室外 AHT20 | 无法初始化室外传感器 |
| TIM3 CH2 输入捕获 (PA7) | `tim.c`: MX_TIM3_Init(), 风扇测速 | 无法测速 |
| TIM3 全局中断 NVIC | `stm32f1xx_it.c`: TIM3_IRQHandler | 捕获中断不触发 |

如果你需要从 CubeMX 重新生成代码, 必须先补充以上配置。详见 `Doc/08_新增文件导入Keil_CubeMX指南.md`。
