## 二、硬件变更说明

### 2.1 四线风扇接线方式

```
         12V ──────────────────────┐
                                   │
                              ┌────┴────┐
                              │  4线风扇  │
                              │  12V DC  │
                              │  红=12V   │──── 直接接 12V (不经过 MOS!)
                              │  黑=GND   │──── 公共 GND
                              │  黄=TACH  │──→ PA7 (TIM3_CH2 输入捕获, 不变)
                              │  蓝=PWM   │──← PA8 (TIM1_CH1 PWM 输出, 25kHz)
                              └──────────┘
```

### 2.2 与3线方案的关键区别

| 对比项 | 3线 + LR7843 MOS | 4线 (内置驱动) |
|--------|-------------------|----------------|
| 12V 供电通断 | MOS 管控制 | **12V 直连, 风扇常供电** |
| PWM 控制方式 | PWM→MOS 栅极→控制 12V 通断 | **PWM→风扇内置 FET→控制内部线圈** |
| PWM 电平 | 3.3V (MOS 兼容) | **3.3V / 5V** (4线风扇有内部上拉到 5V, 3.3V 兼容[^1]) |
| 0% PWM 时 | 风扇完全断电停转 | **风扇低速运转 (约 30% 最低转速)** |
| 100% PWM 时 | 全速 | **全速** |
| 续流二极管 | **需要** (MOS 通断感性负载) | **不需要** (内置驱动) |
| LR7843 模块 | **需要** | **不需要 (移除)** |

[^1]: Intel 4-Wire PWM Fan Spec 定义 PWM 高电平阈值为 2.8V, STM32 3.3V 满足要求。若个别风扇不响应可串联上拉电阻到 5V。

### 2.3 ACS712 电流检测调整

4线风扇 12V 直连, ACS712 **仍然串联在 12V 供电回路中** 检测电流，但由于 4线风扇 0% PWM 时仍有小电流消耗（约 30~50mA），需调整堵转检测阈值：

```
   12V ──→ ACS712 (IP+ IP-) ──→ 风扇 VCC (红线)
                                 风扇 GND (黑线) ──→ GND
```

### 2.4 移除的元件

- ~~LR7843 大功率 MOS 管驱动模块~~ → **移除**
- ~~MOS 栅极 100Ω 串联电阻~~ → **移除**
- ~~MOS G-S 10kΩ 下拉电阻~~ → **移除**
- ~~风扇两端 1N4007 续流二极管~~ → **移除**

> **注意**: 移除 LR7843 后, PA8 的 PWM 信号直接接到风扇蓝线 (PWM 输入)。如风扇 PWM 输入有 5V 上拉, PA8 需串联 **1kΩ 保护电阻**, 防止 5V 倒灌到 STM32 GPIO。

## 三、温度 + 湿度联合调速算法

### 3.1 设计思路

原方案仅依据 **绝对湿度差** (PID 闭环) 决定转速，存在以下局限：
- 热水刚开启时蒸汽未到达传感器, 绝对湿度差为 0, 风扇仅以最低 30% 运行
- 高温环境下蒸发量更大但响应滞后

**最终风扇目标转速不再采用温湿度加权求和，而是由温度通道与湿度通道分别生成风速需求，在热水运行阶段取两者中的较大值作为目标占空比；在延时除湿阶段则回归湿度主导。为避免需求切换带来的风扇抖动，目标占空比在输出前增加升降速斜率限制，实现快速响应与平滑过渡兼顾。**

```
执行链:
  温度需求 (duty_temp)
  湿度需求 (duty_humi)
       ↓
  HOT_WATER: target_duty = max(duty_temp, duty_humi)
  DELAY_STOP: target_duty = 湿度主导, 温度衰减
       ↓
  安全约束 (最小占空比 / 低温保护 / 室外离线保守值)
       ↓
  升降速斜率限制 (升速快, 降速慢)
       ↓
  最终 PWM 输出 → PA8 → 风扇蓝线
```

### 3.2 温度通道: 温度需求计算

温度需求由 **两个子因素** 共同决定:

#### 3.2.1 水温激励 (热水越热 → 蒸汽越多 → 需要更大排风)

```c
// 水温与室温的温差映射
float delta_t = ir_object_temp - ambient_temp;

// 线性映射: 温差 5℃(刚触发) → 30% 占空比
//          温差 25℃(很烫的水) → 90% 占空比
float temp_duty_pct = map_linear(delta_t, 5.0, 25.0, 30.0, 90.0);
float duty_temp = PWM_MAX_DUTY * temp_duty_pct / 100.0;
```

#### 3.2.2 室温修正 (室温越高 → 蒸发越快 → 增加排风)

```c
// 室温修正系数: 18℃ → 0.7, 25℃ → 1.0, 35℃ → 1.3
float temp_factor = 1.0 + (ambient_temp - 25.0) * 0.03;  // 每度 ±3%
temp_factor = clamp(temp_factor, 0.7, 1.3);

// 最终温度需求
duty_temp *= temp_factor;
```

### 3.3 湿度通道: 湿度需求计算 (PID 闭环, 保持不变)

```c
// PV = 室内绝对湿度 (g/m³)
// SP = 室外绝对湿度 (g/m³)
float duty_humi = PID_Compute(&g_pid, indoor_abs_humi);
```

### 3.4 目标占空比决策 — 按状态区分

#### 3.4.1 HOT_WATER 阶段: 取较大值

热水运行阶段, 热源明显、蒸汽明显, **谁更急谁主导** 更合理:

```c
// 温度需求高 → 按温度走; 湿度需求高 → 按湿度走
// 不做平均, 不互相稀释
target_duty = max(duty_temp, duty_humi);
```

> **为什么不用加权求和?** 加权求和会导致任一通道的高需求被另一通道稀释。例如温度分量已要求 80% 排风, 但湿度分量因传感器延迟还在 20%, 加权后只有 `0.4×80+0.6×20=44%`, 远低于实际需要。取 max 则直接输出 80%, 更安全。

#### 3.4.2 DELAY_STOP 阶段: 湿度主导, 温度衰减

延时除湿阶段本质是 "除湿收敛", 不应简单照搬 max:

```c
// 湿度主导, 温度影响逐步减弱
// 温度衰减系数随进入 DELAY_STOP 后的时间递减
float decay = 1.0f - (elapsed_ms / DELAY_STOP_TOTAL_MS);  // 1.0 → 0.0
if (decay < 0.0f) decay = 0.0f;

target_duty = max(duty_humi, duty_temp * decay);
```

> 这样在 DELAY_STOP 初期温度仍有参与, 但随时间推移逐步让位于湿度。如果简单化, 也可以固定比例: `target_duty = max(duty_humi, duty_temp * 0.3f)`。

### 3.5 安全约束 (在斜率限制之前应用)

```c
// 1. 最小占空比
if (target_duty < min_duty) target_duty = min_duty;  // HOT_WATER 最低 30%

// 2. 低温保护上限
if (ambient_temp < 18.0f && target_duty > temp_limit) {
    target_duty = temp_limit;  // 限制 50%
}

// 3. 室外传感器离线 → 保守占空比 (取 max 防止温度通道被覆盖)
if (!g_sys.outdoor_sensor_ok) {
    target_duty = max(target_duty, outdoor_degrade_duty);
}
```

### 3.6 升降速斜率限制 (Slew Rate Limiting)

当温度需求和湿度需求在边界附近来回交叉时, 直接输出 max 值会导致占空比跳变, 表现为风扇忽快忽慢。**斜率限制作用于最终输出, 不是传感器**, 实现快速响应与平滑过渡兼顾。

```c
/* 斜率限制参数 — 每次控制循环 (200ms) 允许的最大变化量 */
#define SLEW_RATE_UP    80    /* 升速: 每 200ms 最多增加 80/2880 ≈ 2.8% */
#define SLEW_RATE_DOWN  30    /* 降速: 每 200ms 最多减少 30/2880 ≈ 1.0% */
```

> **升速快, 降速慢** 的原因:
> - 有蒸汽时要尽快拉风量 → 升速要快
> - 蒸汽刚减弱时不要立刻掉速, 否则会抖 → 降速要慢

```c
/**
 * @brief 对目标占空比施加升降速斜率限制
 * @param target   本次计算的目标占空比
 * @param current  当前实际输出的占空比 (上一轮 final_duty)
 * @return 斜率限制后的最终占空比
 */
static uint16_t SlewRateLimit(float target, uint16_t current)
{
    float diff = target - (float)current;

    if (diff > 0) {
        /* 升速 */
        if (diff > SLEW_RATE_UP) diff = SLEW_RATE_UP;
    } else {
        /* 降速 */
        if (diff < -SLEW_RATE_DOWN) diff = -SLEW_RATE_DOWN;
    }

    float result = (float)current + diff;
    if (result < 0.0f) result = 0.0f;
    if (result > (float)PWM_MAX_DUTY) result = (float)PWM_MAX_DUTY;

    return (uint16_t)result;
}
```

### 3.7 完整执行链

```c
// 完整示例 (HOT_WATER 阶段):
float duty_temp = Calc_TempDuty(delta_t, ambient_temp);      // 温度通道
float duty_humi = PID_Compute(&g_pid, indoor_abs_humi);       // 湿度通道
float target_duty = fmaxf(duty_temp, duty_humi);              // 取较大值

// 安全约束
if (target_duty < min_duty) target_duty = min_duty;
if (ambient_temp < TEMP_LOW_LIMIT && target_duty > temp_limit)
    target_duty = temp_limit;
if (!g_sys.outdoor_sensor_ok)
    target_duty = fmaxf(target_duty, outdoor_degrade_duty);

// 斜率限制 → 最终输出
g_sys.pwm_duty = SlewRateLimit(target_duty, g_sys.pwm_duty);
Fan_SetDuty(g_sys.pwm_duty);
```

### 4.3 `main.c` — 核心调速算法重写

#### 4.3.1 新增辅助函数声明 (在 `USER CODE BEGIN PFP` 区域)

```c
/* 温度 / 湿度双通道调速 */
static float Calc_TempDuty(float delta_t, float ambient_temp);
static uint16_t SlewRateLimit(float target, uint16_t current);
```

#### 4.3.2 新增辅助函数实现 (在 `USER CODE BEGIN 4` 区域)

```c
/* ======================== 温度通道: 需求计算 ======================== */
/**
 * @brief 计算温度通道的风速需求 (水温激励 + 室温修正)
 * @param delta_t       水温 - 室温 温差 (℃)
 * @param ambient_temp  室内温度 (℃)
 * @return 温度通道需求 PWM 值 (0 ~ PWM_MAX_DUTY)
 */
static float Calc_TempDuty(float delta_t, float ambient_temp)
{
    /* 1. 温差线性映射: ΔT → 占空比百分比 */
    float duty_pct;
    if (delta_t <= TEMP_DELTA_LOW) {
        duty_pct = TEMP_DUTY_LOW_PCT;
    } else if (delta_t >= TEMP_DELTA_HIGH) {
        duty_pct = TEMP_DUTY_HIGH_PCT;
    } else {
        /* 线性插值 */
        duty_pct = TEMP_DUTY_LOW_PCT +
                   (delta_t - TEMP_DELTA_LOW) *
                   (TEMP_DUTY_HIGH_PCT - TEMP_DUTY_LOW_PCT) /
                   (TEMP_DELTA_HIGH - TEMP_DELTA_LOW);
    }

    float duty = (float)PWM_MAX_DUTY * duty_pct / 100.0f;

    /* 2. 室温修正: 高温环境蒸发更猛 → 增大排风 */
    float correction = 1.0f + (ambient_temp - TEMP_CORRECTION_CENTER) * TEMP_CORRECTION_RATE;
    if (correction < TEMP_CORRECTION_MIN) correction = TEMP_CORRECTION_MIN;
    if (correction > TEMP_CORRECTION_MAX) correction = TEMP_CORRECTION_MAX;

    duty *= correction;

    /* 3. 钳位 */
    if (duty < 0.0f) duty = 0.0f;
    if (duty > (float)PWM_MAX_DUTY) duty = (float)PWM_MAX_DUTY;

    return duty;
}

/* ======================== 升降速斜率限制 ======================== */
/**
 * @brief 对目标占空比施加升降速斜率限制, 避免需求切换导致风扇抖动
 *
 * 斜率限制作用于 "输出端", 不是传感器。
 * 系统中已有温度滑动平均、ADC 均值、PID 积分/微分等传感器端滤波,
 * 这里只对最终 PWM 目标值做变化率约束。
 *
 * @param target   本次计算的目标占空比 (经过安全约束后)
 * @param current  当前实际输出的占空比 (上一轮 final_duty)
 * @return 斜率限制后的最终占空比
 */
static uint16_t SlewRateLimit(float target, uint16_t current)
{
    float diff = target - (float)current;

    if (diff > 0) {
        /* 升速: 有蒸汽时要尽快拉风量 → 允许较大步进 */
        if (diff > SLEW_RATE_UP) diff = SLEW_RATE_UP;
    } else {
        /* 降速: 蒸汽刚减弱时不要立刻掉速, 否则会抖 → 限制较小步进 */
        if (diff < -SLEW_RATE_DOWN) diff = -SLEW_RATE_DOWN;
    }

    float result = (float)current + diff;
    if (result < 0.0f) result = 0.0f;
    if (result > (float)PWM_MAX_DUTY) result = (float)PWM_MAX_DUTY;

    return (uint16_t)result;
}
```

#### 4.3.4 修改 `App_ExecuteOutput()` — DELAY_STOP 状态

DELAY_STOP 本质是 "除湿收敛", **不简单照搬 max**, 而是让湿度主导, 温度通道随时间衰减:

```diff
     case SYS_STATE_DELAY_STOP: {
+        /* --- 温度通道 (热水已停, 温差逐渐缩小) --- */
+        float delta_t = g_sys.ir_object_temp - g_sys.ambient_temp;
+        float duty_temp = Calc_TempDuty(delta_t, g_sys.ambient_temp);
+
+        /* --- 湿度通道 --- */
+        float duty_humi;
         if (!g_sys.outdoor_sensor_ok) {
-            g_sys.pwm_duty = outdoor_degrade_duty;
+            duty_humi = (float)outdoor_degrade_duty;
         } else {
-            float pid_out = PID_Compute(&g_pid, g_sys.indoor_abs_humi);
-            g_sys.pwm_duty = (uint16_t)pid_out;
+            duty_humi = PID_Compute(&g_pid, g_sys.indoor_abs_humi);
         }
+
+        /* --- 湿度主导, 温度衰减参与 --- */
+        /* 温度衰减: 随进入 DELAY_STOP 后时间递减, 逐步让位于湿度 */
+        float decay = 1.0f - ((float)(HAL_GetTick() - delay_stop_enter_tick)
+                              / (float)DELAY_STOP_TOTAL_MS);
+        if (decay < 0.0f) decay = 0.0f;
+        float duty_temp_decayed = duty_temp * decay;
+
+        /* 取湿度需求与衰减后温度需求的较大值 */
+        float target_duty = (duty_humi > duty_temp_decayed) ? duty_humi : duty_temp_decayed;
+
+        /* --- 安全约束 --- */
+        if (target_duty < min_duty / 2) target_duty = min_duty / 2;
         if (g_sys.ambient_temp < TEMP_LOW_LIMIT && target_duty > temp_limit) {
             target_duty = temp_limit;
         }
+
+        /* --- 斜率限制 → 最终输出 --- */
+        g_sys.pwm_duty = SlewRateLimit(target_duty, g_sys.pwm_duty);
         Fan_SetDuty(g_sys.pwm_duty);
         LED_Set(true, true, false);
         Buzzer_Off();
         break;
     }
```

> **注意**: 需要在进入 DELAY_STOP 状态时记录 `delay_stop_enter_tick = HAL_GetTick()`，用于计算衰减系数。

#### 4.3.5 修改 `Fan_Stop()` — 4线风扇完全停转处理

4线风扇在 PWM=0% 时仍会低速运转 (约 30% 转速)。要完全停转需要将 PWM 信号拉高到 100% 或使用特殊的低电平保持。根据 Intel 4-Wire PWM Fan Spec, **PWM=0% 时风扇应以最低速运转, 拔掉 PWM 线风扇全速运转**。

如果需要 IDLE 状态下完全静音, 有两种方案：

**方案 A: 接受低速运转 (推荐, 无需代码改动)**
```c
// 不需要修改 Fan_Stop(), 0% PWM → 风扇低速运转 → IDLE 时微风
static void Fan_Stop(void)
{
    __HAL_TIM_SET_COMPARE(&htim1, TIM_CHANNEL_1, 0);
    g_sys.fan_running = false;
}
```

**方案 B: 切断 12V 供电实现完全停转 (需额外硬件)**

如果要求 IDLE 完全无风, 则保留 LR7843 MOS 在 12V 回路中做总开关, 使用一个额外 GPIO（如 PB3）控制:
```c
// 需要额外 GPIO 控制 MOS 总开关
static void Fan_Stop(void)
{
    __HAL_TIM_SET_COMPARE(&htim1, TIM_CHANNEL_1, 0);
    HAL_GPIO_WritePin(FAN_POWER_GPIO_Port, FAN_POWER_Pin, GPIO_PIN_RESET); // 断开 12V
    g_sys.fan_running = false;
}

static void Fan_SetDuty(uint16_t duty)
{
    if (duty > PWM_MAX_DUTY) duty = PWM_MAX_DUTY;
    HAL_GPIO_WritePin(FAN_POWER_GPIO_Port, FAN_POWER_Pin, GPIO_PIN_SET);   // 接通 12V
    __HAL_TIM_SET_COMPARE(&htim1, TIM_CHANNEL_1, duty);
    g_sys.fan_running = (duty > 0);
}
```

> **建议使用方案 A**, 因为 4线风扇低速运转噪音极低, 且保持微风有助于传感器灵敏检测。

## 九、注意事项与风险

### 9.1 风险项

| 风险 | 概率 | 缓解措施 |
|------|------|----------|
| 3.3V PWM 不被风扇识别 | 低 | PA8 串联 1KΩ + 10KΩ 上拉到 5V |
| 4线风扇 0% PWM 仍有噪音 | 中 | 可接受; 或用方案 B 断电 |
| 温度/湿度需求交叉导致风扇抖动 | 中 | **升降速斜率限制** 平滑输出, 升速快降速慢 |
| PID 积分与温度通道冲突导致振荡 | 低 | Anti-Windup 已有; max 策略下 PID 独立运行, 不被稀释 |
| 风扇型号不同导致 TACH 频率不同 | 低 | 通用标准: 每转 2 脉冲 |

### 9.2 编码注意

1. **浮点运算**: `Calc_TempDuty` 和 `SlewRateLimit` 使用 `float`, 确保 Keil 勾选了 **Use MicroLIB**
2. **斜率参数调参**: `SLEW_RATE_UP` 和 `SLEW_RATE_DOWN` 可通过串口调参或 Flash 存储, 方便赛场微调体感
3. **DELAY_STOP 衰减**: 进入 DELAY_STOP 时需记录 `delay_stop_enter_tick`, 用于温度衰减系数计算
4. **兼容性**: 如果比赛时只拿到 3线风扇, 只需在 `bsp_config.h` 中 `#define USE_3WIRE_FAN 1` 走原有逻辑即可
